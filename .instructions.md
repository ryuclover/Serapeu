## 🎯 Visão Geral do Projeto

**Serapeu** é uma plataforma de compartilhamento de conhecimento comunitária baseada em tutoriais passo-a-passo. Usuários podem criar, buscar, comentar e salvar tutoriais em categorias como Tecnologia, Culinária, Casa e Jardim e Estilo de Vida.

**Stack:**
- Frontend: Next.js 16 + React 19 + TypeScript + Tailwind CSS
- UI: Radix UI (26+ componentes)
- Backend: Supabase (Auth, Database, Real-time)
- Validação: Zod + React Hook Form
- State: Context API (AuthContext)

---

## ⚙️ Padrões Obrigatórios

### Code Style
- **TypeScript** em todos os arquivos (sem `any`)
- **Naming**: camelCase para variáveis/funções, PascalCase para componentes
- **Imports**: agrupar por ordem (React → libs → local)
- **Componentes**: usar `"use client"` apenas quando necessário

### UI Components
- Usar componentes de `/components/ui/` que já estão wrappados do Radix UI
- Aplicar estilos com Tailwind (`className`)
- Seguir tema com variáveis de cor: `bg-background`, `text-foreground`, etc
- Support dark mode sempre

### Database (Supabase)
- Usar tipos TypeScript para todas as queries
- Implementar validação com Zod antes de salvar
- Adicionar timestamps (`created_at`, `updated_at`) em toda tabela
- Usar soft deletes quando apropriado

### Forms
- Validar com Zod no client (React Hook Form)
- Validar também no backend antes de salvar
- Mostrar erros com Radix UI Alert ou Sonner toast
- Loading states durante requisições

### Error Handling
- Capturar erros Supabase e mostrar mensagens amigáveis
- Log de erros em produção
- Páginas de erro: `error.tsx`, `not-found.tsx`
- Usar Sonner toast para feedback

## 🔄 Fluxo de Autenticação

1. Usuário submete email/senha em `/entrar` ou `/registrar`
2. Supabase autentica e retorna session
3. `AuthContext` armazena user info em estado global
4. Componentes lêem user via `useAuth()` hook
5. Rotas protegidas verificam `user !== null`
6. Logout limpa session e redireciona para home
